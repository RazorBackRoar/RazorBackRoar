## 2026-07-24 - [Fix maintainer contact instructions to prevent public issue creation]

**Vulnerability:** Users were instructed to report vulnerabilities by messaging `@RazorBackRoar` on GitHub directly. Since GitHub doesn't support direct messages, users might resort to opening public issues containing sensitive security details.
**Learning:** Security reporting documentation must always direct users to functional, private channels like an email address or a dedicated private vulnerability reporting form.
**Prevention:** Always verify contact methods exist and can receive private messages before publishing them in security documentation.
